# 米糕记账 PWA 发布到 GitHub Pages

下面的方法不需要 Mac、不需要 Xcode、不需要 Apple Developer 年费，也不需要每 7 天重新安装。只要有一个 GitHub 账号，就可以用浏览器完成。

## 零费用方案的边界

当前采用 GitHub Pages 的公开仓库方案：GitHub 官方说明公开仓库可以使用 GitHub Pages 和 HTTPS，不收 Apple 相关费用。它是本项目目前最接近“长期完全免费”的方案。

但 `github.io` 属于境外站点，国内不同运营商的访问情况可能不同，我不能保证每一部不挂 VPN 的 iPhone 都能稳定打开。项目代码本身不依赖 VPN；如果你的手机能打开发布网址，首次加载完成后，页面资源会由 Service Worker 缓存，记账核心功能可以在没有网络时继续使用。以后要接收网页更新，仍需要偶尔联网访问一次网址。

腾讯云等国内云服务的静态托管更适合国内网络，但目前公开的免费额度可能是限时或限量的，不把它列为“永久免费”方案。Gitee Pages 的官方帮助页也标注过功能调整/下线，因此不作为当前唯一方案。

## 第一次发布

1. 在电脑浏览器打开 [GitHub](https://github.com/) 并登录。
2. 点击右上角 `+`，选择 `New repository`。
3. 仓库名可以填写 `bookkeeping2607`，选择 `Public`，点击 `Create repository`。
4. 打开本地文件夹 `项目/2607记账App-Web`，把里面的所有文件和 `assets` 文件夹上传到仓库最外层。上传后，`index.html` 应该直接位于仓库首页，不要再套一层 `2607记账App-Web` 文件夹。
5. 在仓库页面打开 `Settings` → `Pages`。
6. 在 `Build and deployment` 中选择 `Deploy from a branch`，分支选择 `main`，目录选择 `/(root)`，点击 `Save`。
7. 等待一两分钟，刷新 `Pages` 页面，就会看到网址：

   `https://你的GitHub用户名.github.io/bookkeeping2607/`

8. 在 iPhone 上用 Safari 打开这个网址，确认能正常显示后等待几秒，让离线缓存完成，再点击分享按钮 → `添加到主屏幕`，名称填 `米糕记账`。

如果国内网络打不开这个网址，不要在 iPhone 上反复刷新；这属于托管域名的网络可达性问题，不是记账代码问题。此时仍可以保留 GitHub 作为代码备份，之后再选择有明确长期免费政策的国内托管方式。

## 日后更新

以后只改网页功能时，进入同一个 GitHub 仓库，上传或替换对应文件，等待 Pages 自动重新发布即可。不要更换仓库网址，也不要让代码改动 `bookkeeping2607.pwa.state` 这个存储键；这样后续更新可以继续读取同一个使用环境里的本机已有账单。

调试期如果主屏幕 Web App 没有马上显示新功能，可以进入“我的 → 当前版本”，点击“刷新到最新版本”。这个操作只清理网页代码缓存并重新加载，不会删除账单或云备份配置。

建议每次更新前保留一份本地项目文件夹。网页更新本身不会清除账单，但清除 Safari 网站数据、使用无痕模式、换手机或设备损坏都可能导致本机账单丢失；现在可以在“我的 → 数据管理”导出 JSON 备份，并在新设备导入。

网页打开或保存账单后会在本机自动更新一份辅助快照；如果浏览器的主存储临时损坏，米糕会尝试从快照恢复。由于 iOS 网页不能在关闭时无感写入“文件”或 iCloud 云盘，这份快照不能替代换手机或清除网站数据前的 JSON 导出。

## 可选云备份

如果要长期使用，可以部署 `cloudflare/` 目录里的 Worker 草稿。它采用“手机号账号 + 同步密码/PIN”：

1. 手机号只作为账号标识，方便记住。
2. 同步密码/PIN 在浏览器本机参与加密，不能忘。
3. 云端只保存加密后的账单备份，不保存明文账单。
4. 打开时会自动从云端拉取并合并；保存账单后会自动延迟上传。
5. “我的 → 云备份”里的“立即同步”只是兜底按钮，日常不需要手动操作。
6. 同一个手机号只能对应一套同步密码；如果输入另一个 PIN，会提示同步密码不匹配。

这个方案不使用短信验证码，也不依赖微信登录，因此没有短信费用和微信开放平台审核成本。

## 主屏幕 Web App 作为长期入口

发布后，长期固定从 iPhone 主屏幕“米糕记账”图标打开。这样会进入独立 Web App 视窗，不显示 Safari 地址栏，也会持续使用同一套 Web App 本地账单。

Safari 只作为首次安装、导出旧账单或排查发布网址时使用。不要把 Safari 地址栏里的页面当成另一个日常账本。

### 快速记账

项目支持下面的网址形式直接进入快速记账：

`https://你的GitHub用户名.github.io/bookkeeping2607/?quick=1`

但 iPhone“快捷指令”里的“打开 URL”通常会进入 Safari，所以不建议把它作为日常快捷入口。现在 `manifest.webmanifest` 已经配置了“快速记账”Web App 快捷项；如果你的 iOS 在主屏幕长按“米糕记账”图标时显示它，优先使用这个入口。否则固定从主屏幕图标打开，再点底部记账按钮。

### 重要的数据存储说明

Apple/WebKit 说明，主屏幕独立 Web App 与 Safari 的 cookies 和网站存储是分开的。快捷指令中的“打开 URL”通常会交给 Safari 处理，所以它可能打开 Safari 里的另一套账单空间，而不是主屏幕图标对应的独立 Web App。

当前项目没有自动云同步，但支持 JSON 导出与导入。为了确保长期账单都在主屏幕 Web App 这一套里，推荐这样处理：

1. 用 Safari 打开发布网址。
2. 添加到主屏幕，名称填 `米糕记账`。
3. 之后日常记账都从主屏幕图标进入。
4. 如果 Safari 里已经有旧账单，在 Safari 那套也开启同一个云备份账号；再回到主屏幕 Web App 开启同一账号，打开时会自动合并。

应用内不会提供“清空本机全部账单”按钮，避免误触导致长期数据丢失。换手机、清理网站数据或大版本更新前，先导出一份 JSON 备份到“文件”或 iCloud 云盘。

## 关于授权和流量

本项目本身是自己编写的 HTML、CSS 和 JavaScript，只依赖项目内的米糕素材。发布到 GitHub Pages 只上传网页代码和图片，不上传账单数据；日常记账主要在 iPhone 本地运行，不会产生 GB 级别的持续下载流量。
